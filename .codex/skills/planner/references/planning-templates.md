# Planning Templates

Use these templates only when the default output contract in `SKILL.md` is not enough.

## Compact Plan

```markdown
Muc tieu: ...
Pham vi: ...
Rang buoc: ...
Gia dinh: ...

1. Milestone A
Done when: ...

2. Milestone B
Done when: ...

Dependencies: ...
Rui ro chinh: ...
Kiem chung: ...
Project state: ...
Task breakdown: ...
Demo / handover: ...
Buoc dau tien: ...
```

## Multi-Phase Plan

```markdown
Muc tieu: ...
Pham vi: ...
Rang buoc: ...
Unknowns can resolve: ...

Phase 1. Discovery
Done when: key unknowns resolved

Phase 2. Design
Done when: approach chosen and interfaces agreed

Phase 3. Implementation
Done when: code/config complete

Phase 4. Validation
Done when: tests/checks/security review pass

Phase 5. Release
Done when: rollout and rollback steps are ready or executed

Critical path: ...
Parallel tracks: ...
Approval gates: ...
Project state: ...
Task breakdown: ...
Demo / handover: ...
Buoc dau tien: ...
```

## Review Questions

Use this quick review before sending the plan:

- Does each milestone produce an observable outcome?
- Did you separate unknowns from execution steps?
- Did you avoid fake precision where estimates are weak?
- Did you add rollback thinking for risky changes?
- Did you make project state, task breakdown, demo, and handover explicit when the work is freelancer-facing?
