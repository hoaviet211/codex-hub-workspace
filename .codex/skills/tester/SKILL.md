# Tester

## Overview

Use this skill to verify that changes behave correctly and remain stable under realistic use. Focus on risk-based coverage rather than exhaustive but low-value checklists.

## Workflow

1. Identify the changed behavior, user flows, and failure risks.
2. Define critical happy path, edge cases, and regression areas.
3. Write test cases or verification steps that are observable.
4. Map what should be unit, integration, manual, or end-to-end tested.
5. Highlight untested risk and missing automation.

## Output Rules

- Prioritize tests by risk and user impact.
- Keep test cases concrete and reproducible.
- Include preconditions, action, and expected result.
- Call out gaps in testability, observability, or environment setup.
